На этой неделе я изучал CI/CD (Непрерывная интеграция/Непрерывное развёртывание) с использованием GitHub Actions.

## Что такое CI/CD?

CI/CD автоматизирует процесс тестирования и развёртывания кода. Вместо ручной сборки и загрузки изменений, CI/CD делает это автоматически при каждом push в репозиторий.

## Что я узнал

Я настроил рабочий процесс GitHub Actions, который:
- Автоматически собирает сайт Hugo
- Запускает тесты (если они есть)
- Развёртывает на GitHub Pages при каждом push

Это был захватывающий опыт, теперь я понимаю, как работает современная разработка ПО с автоматизированными конвейерами.

## Сложности

Настройка рабочего процесса заняла некоторое время. Мне нужно было понять синтаксис YAML и конфигурацию GitHub. Также возникли проблемы с разрешениями, потребовавшие генерации токена GitHub.
